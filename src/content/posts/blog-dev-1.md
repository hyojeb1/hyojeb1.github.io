# 시작하며: 왜 직접 블로그를 만들었나?

어느 날 문득 이런 생각이 들었습니다. "내 포트폴리오를 보여줄 완벽한 놀이터가 필요해." 
그래서 결심했습니다. 나만의 커스텀 기능이 듬뿍 담긴, 100% 내 통제하에 있는 블로그를 직접 만들기로요.

## 🏃 플랫폼 이주: 기존 플랫폼들의 한계

이전에 [Velog(hyojeb1)](https://velog.io/@hyojeb1)에서 나름 열심히 글을 썼고, Notion도 적극적으로 활용해 봤습니다. 하지만 쓰면 쓸수록 채워지지 않는 갈증이 있었습니다.

:::important
**내가 원했던 것:** WebGL이나 직접 만든 게임 엔진 결과물을 브라우저 위에서 동적으로 렌더링해서 보여주기
:::

Notion은 너무 그들의 서버와 시스템에 종속되어 있어서 제가 원하는 수준의 커스터마이징이 불가능했습니다. (노션 이 자식들, 돈에 미쳤구먼... 💸) Velog 역시 훌륭한 플랫폼이지만, 정해진 틀 안에서만 글을 써야 한다는 답답함이 컸죠.

## 🛠️ 기술 스택 선정: 왜 하필 Astro인가?

수많은 정적 사이트 생성기(Jekyll, Hugo, Next.js 등)를 두고 고민하던 중, 든든한 조력자 제미나이(Gemini)가 1순위로 추천해 준 것이 바로 **Astro**였습니다.

> 학부 때 쓰던 React를 안 쓰고 Astro를 선택한 이유? 
> 웹 개발에서 손 뗀 지 너무 오래돼서 제미나이에 전적으로 의존할 수밖에 없었기 때문입니다. (압도적 감사...)

Astro는 불필요한 자바스크립트를 덜어내어 렌더링 속도가 미치도록 빠르고, 마크다운(Markdown) 기반 블로그를 구축하는 데 이보다 더 편할 수 없는 생태계를 갖추고 있었습니다. 저는 여기에 59가지 테마를 지원하는 **MultiTerm** 템플릿을 얹어 저만의 입맛대로 커스텀하기 시작했습니다.

## 🎢 배보다 배꼽이 커지는데?

"블로그 만드느라 정작 개발 공부할 시간이 없는 거 아니야?" 
맞습니다. 배보다 배꼽이 커지고 있는 게 사실입니다. 하지만 평소에 늘 꿔왔던 꿈이기에 멈출 수가 없네요.

:::swallow
무엇보다... **일단 너무 재밌습니다!** 순수 재미 100%!
:::

딸깍딸깍, 코드를 고치고 터미널 감성의 화면이 렌더링 되는 걸 볼 때마다 엄청난 희열을 느낍니다. *노션 따잇! 벨로그 따잇!* 그리고 솔직히 말해서... Velog 이 자식들, 내 피 같은 글에 광고를 못 넣게 하잖아요? 반드시 이 블로그를 키워서 돈을 벌어주마! (물론 `.github.io` 도메인은 구글 애드센스 승인이 어렵다는 제미나이의 뼈아픈 경고가 있었지만... 그건 그때 가서 생각하기로 합니다.)

## ☁️ GitHub Pages를 선택한 이유

정말 단순합니다. **서버가 없어서(무료라서)** 입니다.
물론 아예 없는 건 아니고 Microsoft의 인프라를 빌려 쓰는 것이지만, 백엔드 서버 유지비나 트래픽 걱정 없이 온전히 프론트엔드와 콘텐츠에만 집중할 수 있다는 건 엄청난 메리트입니다.

## 🔮 나만의 무기: 커스텀 마크다운 환경 구축

이 블로그의 핵심은 제가 평소 글을 쓸 때 애용하는 **Obsidian(옵시디언)** 앱과의 완벽한 연동입니다. 이를 위해 Astro 파이프라인에 커스텀 플러그인을 직접 깎아 넣었습니다.

### Obsidian Markdown Support
옵시디언에서 작성한 마크다운 문법을 Astro 환경에서 완벽하게 렌더링하도록 커스텀 Remark/Rehype 플러그인을 적용했습니다.

* `remarkObsidianCallout`: `> [!info]` 형태의 옵시디언식 콜아웃(Admonitions)을 예쁜 UI 박스로 변환합니다.
* `remarkObsidianWikilinkHighlight`: `[[link]]` 형태의 위키링크를 일반 `<a>` 태그로, `==text==`를 형광펜 `<mark>` 태그로 변환합니다.
* **Strict Tag Routing:** 프론트매터의 태그 규칙을 수정하여 `foo:goo` 같은 콜론(:) 기반 태그 라우팅이 정상적으로 동작하도록 보장하는 로직을 추가했습니다.

```javascript title="astro.config.mjs"
// 이제 옵시디언에서 쓰던 방식 그대로 블로그에 렌더링됩니다.
import { remarkObsidianCallout } from './src/plugins/remark-obsidian-callout';

export default defineConfig({
  markdown: {
    remarkPlugins: [remarkObsidianCallout],
  },
});

```

앞으로 이 블로그에서 오로라 엔진 개발기부터 자전거 국토종주 후기까지 다양한 이야기들을 쏟아낼 예정입니다. 지켜봐 주세요!
